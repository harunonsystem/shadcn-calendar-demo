# 国際化（i18n）設計ガイド

## 概要

このプロジェクトの国際化は、**Single Source of Truth**の原則に基づいて設計されています。

## ディレクトリ構成

```
src/lib/i18n/
├── config.ts       # 言語設定の定義（Language型、デフォルト言語）
├── translations.ts # 翻訳データ
└── index.ts        # エクスポート・カスタムフック
```

## ファイル役割

### `config.ts` - 言語設定

```typescript
// サポート言語の定義（ここに追加するだけで新言語対応）
export const supportedLanguages = ['ja', 'en'] as const

// Language型は配列から自動導出（手動管理不要）
export type Language = (typeof supportedLanguages)[number]

// デフォルト言語（変更はここだけ）
export const defaultLanguage: Language = 'ja'
```

### `translations.ts` - 翻訳データ

各言語の翻訳テキストを定義。構造:

```typescript
export const translations = {
  ja: {
    today: '今日',
    allDay: '終日',
    addEventModal: { title: '新規イベント', ... },
    locale: 'ja-JP',          // toLocaleDateString用
    formats: { monthYear: '{year}年 {month}', ... },
    units: { minutes: '分', hours: '時間' },
    ...
  },
  en: { ... }
}
```

### `index.ts` - エクスポート

```typescript
// Reactコンポーネント用フック（Jotai atom連携）
export const useTranslation = ()

// 関数内で使用（非React）
export const getTranslation = (language: Language)

// ユーティリティ
export const getWeekDayName = (language, dayIndex)
export const getWeekDayLabels = (language, firstDayOfWeek)

// 型・設定の再エクスポート
export type { Language } from './config'
export { defaultLanguage, supportedLanguages } from './config'
```

## 使い方

### Reactコンポーネント内

```tsx
import { useTranslation } from '@/lib/i18n'

function MyComponent() {
  const t = useTranslation()
  return <button>{t.addEvent}</button>
}
```

### 言語をpropsで受け取る場合

```tsx
import { getTranslation } from '@/lib/i18n'

function MyComponent({ language }: { language: Language }) {
  const t = getTranslation(language)
  return <div>{t.today}</div>
}
```

### 非Reactコンテキスト（ユーティリティ関数など）

```typescript
import { getTranslation, type Language } from '@/lib/i18n'

export const formatDuration = (ms: number, language: Language) => {
  const t = getTranslation(language)
  return `${Math.floor(ms / 60000)}${t.units.minutes}`
}
```

## 新しい言語を追加する手順

### 1. `config.ts` に言語コードを追加

```typescript
export const supportedLanguages = ['ja', 'en', 'fr'] as const
//                                              ^^^^追加
```

### 2. `translations.ts` に翻訳を追加

```typescript
export const translations = {
  ja: { ... },
  en: { ... },
  fr: {                    // 追加
    today: "Aujourd'hui",
    allDay: 'Toute la journée',
    locale: 'fr-FR',
    // ... 他のすべてのキー
  }
}
```

**これだけで完了！** 型は自動で `'ja' | 'en' | 'fr'` になります。

---

## デフォルト言語を変更する手順

`config.ts` の1行を変更するだけ:

```typescript
export const defaultLanguage: Language = 'en'  // 'ja' → 'en'
```

---

## 禁止事項

### ❌ コンポーネント内での言語ハードコード

```tsx
// NG: 言語追加時に全箇所修正が必要
const label = language === 'ja' ? '今日' : 'Today'
```

### ✅ 翻訳システムを使用

```tsx
// OK: 翻訳追加だけで対応完了
const t = getTranslation(language)
const label = t.today
```

---

## 翻訳キーを追加する手順

1. `translations.ts` の **すべての言語** に同じキーを追加
2. `Translations` 型に追加（型安全性のため）

```typescript
// translations.ts
export const translations = {
  ja: { newKey: '新しいテキスト', ... },
  en: { newKey: 'New text', ... },
}

// Translations型にも追加
export type Translations = {
  readonly newKey: string
  // ...
}
```
